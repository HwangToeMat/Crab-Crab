# [세미나] Gemini CLI & 꽃게팀: LLM 에이전트 기반 자율 개발 프로세스 구축 사례

## 1. 프로젝트 개요 (Project Overview)
본 프로젝트는 **Gemini CLI**를 활용하여 기획부터 개발, 검증, 고도화까지 전 과정을 자율적으로 수행하는 멀티 에이전트 시스템 **'꽃게팀(CrabTeam)'**의 워크플로우를 실험하고 구축한 사례입니다.

- **핵심 목표**: 개발자의 개입을 최소화하고, LLM 에이전트들이 각자의 전문성(기획, 디자인, 설계, 개발, QA)을 바탕으로 시장 트렌드에 맞는 서비스를 완결성 있게 인도하는 것.
- **활용 도구**: Gemini CLI, Python(FastAPI), JavaScript, Git.

---

## 2. 꽃게팀(CrabTeam) 아키텍처 및 에이전트 셋팅

단순한 코드 생성을 넘어 실제 제품 개발 조직의 문화를 이식하기 위해 **역할 기반 에이전트(Role-based Agents)**를 구성했습니다.

### 2.1 주요 에이전트 및 역할
| 에이전트명 | 역할 (Role) | 도입 이유 |
| :--- | :--- | :--- |
| **Trend Analyst** | 시장 트렌드 분석 및 기획 소스 제공 | 데이터 기반의 '팔릴 만한' 서비스 아이디어를 도출하기 위함. |
| **Service Planner** | PRD(제품 요구사항 문서) 작성 | 목표 고객(User Persona)과 핵심 기능(MVP)을 명확히 정의하여 개발 방향성 고정. |
| **Tech Architect** | 기술 스택 선정 및 DB 스키마 설계 | 확장 가능하고 유지보수가 용이한 기술 구조를 사전에 정의. |
| **UI/UX Designer** | 디자인 토큰 및 사용자 여정(User Journey) 설계 | 브랜드 정체성 확립 및 일관된 사용자 경험 제공. |
| **Global Validator** | 전 과정 결과물 검증 및 논리 체크 | 각 에이전트 간의 결과물이 상충되지 않는지 감시하는 '교차 검증' 역할. |

## 3. 개발 프로세스 (The 4-Phase Workflow)

### Phase 1: 자율 기획 (Autonomous Planning)
- **수행**: 트렌드 분석가와 서비스 기획자가 2026년 한국 시장의 키워드인 **'필코노미(Feelconomy)'**와 **'1.5인 가구'**를 포착.
- **결과**: 감정 기반 소셜 큐레이션 서비스 **"꽃게메이트(Crab-Mate)"** 컨셉 도출.

### Phase 2: 구현 (Implementation)
- **백엔드**: FastAPI를 활용해 가볍고 빠른 API 서버 구축. AI 무드 분석 로직 포함.
- **프론트엔드**: Vanilla JS와 CSS3를 활용하여 디자인 토큰(Crab Orange 등)을 반영한 모던한 UI 구현.

### Phase 3: 10단계 검증 루프 (10-Cycle Verification)
- **왜 했는가?**: 개발 직후의 제품은 가설일 뿐입니다. 이를 실제 사용자의 시각에서 검증하기 위해 **10인의 가상 페르소나(학생, 직장인, 개발자, 은퇴자 등)**를 투입했습니다.
- **성과**: 로딩 상태 미비, UI 간격 불일치, 개인정보 보안 우려 등 실제 유저가 느낄 법한 페인 포인트를 10회에 걸쳐 수정하여 제품 완성도를 극대화했습니다.

## 4. 고도화 단계: 진화 감독관(Evolution Director) 도입

오늘의 가장 핵심적인 실험 중 하나는 **'런칭 이후의 자율적 성장'**이었습니다. 이를 위해 신규 에이전트인 **진화 감독관(Evolution Director)**을 추가했습니다.

### 도입 배경 (Why Evolution?)
서비스는 배포 후 방치되는 순간 가치를 잃습니다. 고객의 실제 의견과 경쟁사의 움직임, 기술 트렌드 변화를 수집하여 **서비스를 스스로 업그레이드하는 주체**가 필요했습니다.

### 진화 감독관의 4대 데이터 소스
1. **고객 의견**: 페르소나 피드백 분석.
2. **내부 전문가**: 아키텍트와 디자이너의 기술적/미적 업그레이드 제안.
3. **인터넷 리서치**: 최신 AI 트렌드 반영.
4. **경쟁사 분석**: 타사 서비스의 강점 흡수.

### 진화 결과 (Cycles 1~10)
- **AI 고도화**: 단순 분석을 넘어 '불안', '복합 감정' 등 세분화된 감정 카테고리 지원.
- **데이터 자산화**: 사용자별 감정 흐름(Mood History) 시각화 기능 추가.
- **소셜 연대**: 동일한 기분을 느끼는 유저 수 표시 및 익명 응원(Cheer-up) 기능으로 리텐션 강화.
- **성능 최적화**: API 캐싱 시뮬레이션을 통해 응답 속도 개선.

## 5. 결론 및 시사점 (Takeaways)

1. **에이전트 협업의 힘**: 혼자 코드를 짜는 것보다, 기획자/디자이너/아키텍트의 페르소나를 가진 에이전트들이 서로의 결과물을 검증하며 진행할 때 코드의 질과 서비스의 비즈니스적 가치가 훨씬 높았습니다.
2. **반복의 가치**: 10회 이상의 검증 및 진화 루프를 통해 초기 MVP에서 발견하지 못한 수많은 엣지 케이스와 UX 결함을 해결할 수 있었습니다.
3. **자율 개발의 미래**: Gemini CLI와 같은 강력한 인터페이스를 통해, 이제 개발자는 '구현'에만 매몰되지 않고 전체 시스템의 '방향성'과 '에이전트 오케스트레이션'에 집중하는 시대로 진입하고 있습니다.
