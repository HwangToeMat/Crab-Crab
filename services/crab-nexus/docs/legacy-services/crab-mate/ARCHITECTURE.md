# ARCHITECTURE: 꽃게메이트 (Crab-Mate) - 시스템 설계 및 기술 스택

## 1. 기술 스택 (Tech Stack)
### 1.1 Backend
- **Framework**: FastAPI (Python 3.10+) - 고성능 비동기 처리 및 AI 모델 연동의 용이성.
- **Database**: SQLite - MVP 단계의 빠른 개발 및 배포를 위해 선정.
- **AI Library**: `transformers` 또는 `textblob` (간단한 감성 분석용), `nltk`.
- **API Documentation**: Swagger (Auto-generated by FastAPI).

### 1.2 Frontend
- **Framework**: React (Vite) - 빠른 빌드 및 현대적인 컴포넌트 기반 UI 개발.
- **State Management**: React Context API or Zustand.
- **Styling**: Vanilla CSS (Workspace Standard).

## 2. 시스템 아키텍처 (System Architecture)
- **Pattern**: Monolithic for MVP (Scale-out considered via Docker).
- **Flow**:
  1. **Frontend**: 사용자 입력 수신 -> API 요청.
  2. **Backend**: 감성 분석(Sentiment Analysis) 수행 -> 무드 키워드 추출.
  3. **Database**: 해당 무드에 적합한 추천 액티비티 쿼리.
  4. **Backend**: 분석 결과 및 추천 목록 반환.

## 3. 데이터베이스 스키마 (Database Schema)
### 3.1 UserMoods
- `id`: INTEGER (PK)
- `user_id`: STRING
- `mood_text`: TEXT
- `sentiment_score`: FLOAT
- `mood_category`: STRING (Happy, Sad, Angry, Neutral)
- `created_at`: TIMESTAMP

### 3.2 Activities
- `id`: INTEGER (PK)
- `title`: STRING
- `description`: TEXT
- `mood_category`: STRING
- `link`: STRING

## 4. API 엔드포인트 (API Endpoints)
- `POST /api/mood/analyze`: 사용자의 텍스트를 분석하여 무드 반환.
- `GET /api/activities/recommend`: 특정 무드에 기반한 추천 목록 조회.
- `POST /api/matching/request`: 무드 매칭 요청.
- `GET /api/health`: 서비스 상태 체크.

## 5. 인프라 및 배포 (Infrastructure)
- **Containerization**: Docker (Dockerfile included).
- **CI/CD**: GitHub Actions (Potential).
- **Cloud**: AWS Lambda or EC2 (Target).
