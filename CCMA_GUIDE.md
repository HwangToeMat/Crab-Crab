# Crab-Crab Multi-Agent (CCMA) System Guide

Welcome to the **CCMA (Crab-Crab Multi-Agent)** system. This framework is designed to orchestrate 9 specialized agents to collaboratively build and verify IT services.

## System Architecture

The system is structured as follows:

-   **agents/**: Contains persona definitions (Markdown) for each agent. These define their roles, goals, and constraints.
-   **tasks/**: Directory for tracking task status and inputs/outputs.
-   **workflows/**: Contains the `orchestrator.py` script and logic for agent interaction.
-   **outputs/**: Stores the final artifacts generated by the agents (PRDs, code, designs).

## The CCMA Ecosystem

The system consists of the **CrabTeam (꽃게팀)**, who builds the service, and the **Evaluation Group**, who ensures the service meets market standards.

### 1. CrabTeam (꽃게팀) - The Builders
These agents are responsible for the entire development lifecycle:
1.  **Trend Analyst (KR)**: Identifies market opportunities.
2.  **Service Planner (PM)**: Drafts the PRD.
3.  **UI/UX Designer**: Creates the visual identity.
4.  **Tech Architect**: Designs the system architecture.
5.  **Frontend Developer**: Builds the UI.
6.  **Backend Developer**: Builds the core logic.
7.  **DevOps & Infra Engineer**: Manages deployment and cloud.
8.  **QA Specialist**: Ensures quality through testing.
9.  **Growth & Marketing Strategist**: Plans for user acquisition.
10. **Strategy Lead**: Provides high-level direction.
11. **Global Validator**: Ensures cross-agent consistency.

### 2. Evaluation Group - The Critics
These agents simulate the real market and provide feedback for improvement:
12. **User Persona Group**: A diverse set of virtual users (10s to 60s) providing raw feedback.
13. **Consumer Insight Expert**: Synthesizes user feedback into an actionable improvement roadmap for the CrabTeam.

## Mutual Verification Loop

One of the core features of CCMA is the **Mutual Verification Loop**. This ensures that no output is finalized without a critical review from another agent.

### Process:
1.  **Drafting**: Agent A (e.g., PM) creates a draft output (e.g., PRD).
2.  **Verification**: Agent B (e.g., Global Validator or Strategy Lead) reviews the draft.
3.  **Feedback**: If flaws are found, the output is returned to Agent A with actionable feedback.
4.  **Revision**: Agent A revises the work and submits it for a second review.
5.  **Approval**: Once the reviewer is satisfied, the output is finalized and passed to the next agent in the pipeline (e.g., Tech Architect).

## How to Run

1.  Navigate to the `workflows/` directory.
2.  Run the orchestrator: `python orchestrator.py --task "Identify a trend and plan a service"`.
