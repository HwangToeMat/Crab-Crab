# 📋 꽃게팀(CrabTeam) 운영 가이드 (User Manual)

본 문서는 사용자가 꽃게팀 시스템을 효율적으로 운영하고 지시하기 위한 매뉴얼입니다.

## 1. 꽃게팀의 동작 방식
꽃게팀은 **'Lead Architect & Project Manager'** 페르소나를 중심으로 여러 전문 에이전트가 협업하는 자율 팀입니다.

### 에이전트 활용 프로세스
- **기획:** `trend_analyst`가 시장 조사를 하고, `service_planner`가 PRD를 작성합니다.
- **개발:** `tech_architect`가 설계를 잡으면 `backend_dev`와 `frontend_dev`가 실제 코드를 작성합니다.
- **품질:** `qa_specialist`가 10회의 테스트를 수행하고, `global_validator`가 최종 승인합니다.
- **운영:** `devops_infra`가 Docker 환경을 구축하고 배포를 준비합니다.
- **고도화:** `evolution_director`가 V2.0 기능(AI 연동 등)을 추가합니다.

## 2. 서비스 개발 및 개선 과정
1. **아이디어 투입:** 사용자가 간단한 서비스 아이디어를 제시합니다.
2. **Phase 1-2 실행:** 기획과 개발이 동시에 진행되며 Docker 환경까지 구축됩니다.
3. **10회 검증:** 서비스의 안정성을 위해 10번의 반복 테스트와 리포트가 생성됩니다.
4. **최종 통합:** 작업이 완료되면 별도의 PR 없이 `main` 브랜치에 직접 반영됩니다.

## 3. 사용자 지시 가이드
꽃게팀은 특정 키워드(Trigger)를 통해 전체 워크플로우를 가동합니다.

| 지시어 | 설명 |
| :--- | :--- |
| **"꽃게팀 작업개시 [아이디어]"** | Phase 1~4를 자율적으로 완수 (기획부터 최종 보고까지) |
| **"꽃게팀 진화개시 [서비스명]"** | Phase 5를 수행 (V2.0 고도화, AI 연동 등) |
| **"꽃게팀 무한진화개시 [아이디어]"** | **[강력 추천]** Phase 1~5 전 과정을 멈춤 없이 완수 (Full Cycle) |
| **"특정 에이전트 호출 [명령]"** | 특정 에이전트(예: 보안 에이전트)에게만 별도 작업을 지시 |

### 💡 팁
- **무한진화개시:** 처음부터 고도화된 AI 기능까지 포함된 완성형 서비스를 원할 때 사용하세요.
- **자율성 존중:** 꽃게팀은 중간 승인 없이도 스스로 최선의 결정을 내립니다. 완료 후 리포트만 확인하시면 됩니다.
- **피드백:** 결과물이 마음에 들지 않을 경우, `MASTER_GUIDELINE.md`를 수정하라고 지시하거나 특정 단계부터 재작업을 요청하세요.
- **모니터링:** `reports/cycles/` 폴더를 통해 실시간 검증 현황을 파악할 수 있습니다.
