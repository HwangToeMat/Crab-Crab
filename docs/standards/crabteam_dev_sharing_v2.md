# CrabTeam v2.0: AI-Native Autonomous Development Framework

![CrabTeam Logo](crabteam.png)

본 문서는 **꽃게팀(CrabTeam)**이 v2.0 '무한진화' 단계에서 확립한 **AI 에이전트 기반 자율 개발 프로세스**와 **페르소나 설계 전략**을 상세히 공유합니다. 인간 개발자는 코딩을 넘어 '오케스트레이션'과 '아키텍처 가이드라인 수립'에 집중하는 새로운 패러다임을 제안합니다.

---

## 1. AI-Native 개발 철학: Persona-Driven Development (PDD)

단순한 코드 생성을 넘어, 고도화된 소프트웨어를 구축하기 위해 꽃게팀은 **'페르소나 기반 개발(PDD)'** 방법론을 사용합니다. 

### 1.1 지식의 수직적 계층화
- **전략 에이전트**: "무엇을 만들 것인가?" (Why & What)
- **설계 에이전트**: "어떻게 구조화할 것인가?" (How)
- **구현 에이전트**: "실제 코드는 무엇인가?" (Action)
- **검증 에이전트**: "구조와 로직이 완벽한가?" (Validation)

### 1.2 상호 견제와 균형 (Checks & Balances)
에이전트들은 서로의 산출물을 검토합니다. 예를 들어, `Architecture Standardizer`는 `Backend Dev`가 작성한 코드가 글로벌 컨벤션을 위반할 경우 작업을 중단시키고 리팩터링을 강제합니다.

---

## 2. 핵심 에이전트 페르소나 및 역할 (V2.0 최신화)

V2.0 '무한진화'를 위해 새롭게 추가되거나 강화된 페르소나들입니다.

### 2.1 아키텍처 표준화 에이전트 (Architecture Standardizer) - **NEW**
- **역할**: 파편화된 기술 스택을 통합하고 코드 품질의 일관성을 강제합니다.
- **특징**: 중복 코드를 혐오하며, 전체 생태계를 관통하는 단일 진실 공급원(Single Source of Truth)을 유지합니다. 이번 리팩터링에서 BE(Spring Boot)와 FE(React) 통합을 주도했습니다.

### 2.2 진화 감독관 (Evolution Director)
- **역할**: "꽃게팀 무한진화개시" 명령의 수행자입니다.
- **특징**: 단순 버그 수정을 넘어, AI 코치 통합, 신뢰 보증 아키텍처 도입 등 시스템의 '차원'을 높이는 전략적 고도화를 수행합니다.

### 2.3 글로벌 검증자 (Global Validator)
- **역할**: 팀 내 최종 승인권자입니다.
- **특징**: 모든 에이전트의 산출물이 CrabTeam 워크플로우(`CRABTEAM_WORKFLOW.md`)를 준수했는지 논리적으로 검증합니다.

## 3. 자율 개발 워크플로우 (Autonomy Loop)

꽃게팀의 개발은 인간의 개입 없이 아래 5단계 루프를 통해 수행됩니다.

### Phase 1: Research & Discovery
- **에이전트**: Trend Analyst, Service Planner
- **액션**: 기존 코드베이스 스캔, 사용자 니즈 분석, 새로운 시장 기회 포착.

### Phase 2: Strategic Strategy
- **에이전트**: Strategy Lead, Tech Architect, Architecture Standardizer
- **액션**: 마이그레이션 전략 수립, 기술 스택 확정(JDK 25, SB 4.x), 통합 도메인 설계.

### Phase 3: Surgical Execution
- **에이전트**: Backend Dev, Frontend Dev
- **액션**: 표준 컨벤션에 따른 코드 이식. `replace`와 `write_file` 도구를 사용한 정밀한 코드 조작.

### Phase 4: 10-Cycle Verification
- **에이전트**: QA Specialist, User Persona Group
- **액션**: 10회의 자율 개선 루프 가동. 각 루프마다 리포트를 작성하며 스스로 결함을 수정.

### Phase 5: Infinite Scaling (Evolution)
- **에이전트**: Evolution Director, Gemini AI Engine
- **액션**: 지능형 기능(AI Coach, Sentinel) 통합 및 글로벌 통합 포털(Crab-Nexus) 구축.

---

## 4. 에이전트 협업의 핵심: Context & Hooks

에이전트가 완벽하게 일하기 위해 제공되는 데이터 환경입니다.

1. **Foundational Context**: `.gemini/GEMINI.md`와 같은 파일에 팀의 '헌법'과 같은 행동 강령을 명시합니다.
2. **Standardization Hooks**: `SPRING_REACT_CONVENTION.md`와 같은 문서를 에이전트에게 주입하여 기술적 이탈을 방지합니다.
3. **State Management**: `state.json`을 통해 현재 생태계의 모든 서비스 가동 상태를 에이전트끼리 공유합니다.

---

## 5. 결론: 인간의 역할 변화

AI 에이전트 기반 개발 환경에서 인간(Owner)의 역할은 다음과 같이 변화합니다.

- **Before**: 직접 코드 작성, 라이브러리 버전 관리, 수동 테스트.
- **After**: **페르소나 설계(Prompt Engineering)**, **비즈니스 목표 설정**, **최종 가치 판단**.

꽃게팀 v2.0은 AI가 스스로 아키텍처를 리팩터링하고, 수만 라인의 레거시를 정리하며, 통합 포털을 구축하는 과정을 통해 **'자율 개발의 실현 가능성'**을 증명했습니다.
